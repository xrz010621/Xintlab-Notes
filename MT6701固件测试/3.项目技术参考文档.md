# MT6701 磁铁精度测试 — 项目技术参考文档

## 一、项目概述

### 1.1 测试目的

在 EVT v2 电机模块上测试不同直径、厚度的径向充磁圆柱形磁铁对 **MT6701 14 位绝对式磁旋转编码器** 精度的影响：

- 调整磁铁与传感器间距，分别验证 **2mm / 3mm / 4mm / 5mm** 的可用性
- 更换不同规格磁铁（不同直径 × 厚度），评估磁铁尺寸对精度的适配性
- 在电机单元转动范围内（1.5°~71.5°，共 70° 行程）采集精度数据

### 1.2 数据链路拓扑

系统包含**两条独立的物理总线**，APM32 作为中间桥接：

```
┌─────────────────────────────────────────────────────────────────┐
│                        APM32F103CBT6                            │
│                                                                 │
│   ┌─────────────────┐          ┌─────────────────────────────┐  │
│   │   UART Shell     │          │      主控制循环 (10ms)       │  │
│   │  (test_shell.c)  │          │                             │  │
│   │                  │          │  motor_ctrl_get_angle()     │  │
│   │  STATUS ───触发──┼──► Mt6701ReadAngleDeg()               │  │
│   │  MOVE   ───► ProcessCanCommand()                          │  │
│   │                  │          │  s_lastRawCount (缓存)       │  │
│   │  [TS] STATUS     │          │  DebugUartSendRuntimeStatus │  │
│   │  [TS] OK MOVE    │          │   (每 500ms 流式输出)       │  │
│   └────────┬─────────┘          └──────────────┬──────────────┘  │
│            │                                   │                 │
│            │  UART (PA9/PA10)                  │  软件 I²C       │
│            │  115200 8N1                       │  PB10(SCL)      │
│            │  ASCII 命令/回复                   │  PB11(SDA)      │
│            │                                   │                 │
└────────────┼───────────────────────────────────┼─────────────────┘
             │                                   │
             ▼                                   ▼
     ┌──────────────┐                   ┌──────────────────┐
     │   PC (脚本)   │                   │     MT6701       │
     │   pyserial   │                   │   14-bit 编码器   │
     │              │                   │   I²C 地址 0x06   │
     │  发送:       │                   │  寄存器:          │
     │  STATUS\n    │                   │   0x03 ANGLE_H    │
     │  MOVE xxxx\n │                   │   0x04 ANGLE_L    │
     └──────────────┘                   └──────────────────┘
         UART 总线                            I²C 总线
    (PC ↔ APM32 直连)                   (APM32 ↔ MT6701 直连)
```

**两条总线各自独立**：PC 不直接访问 MT6701，所有传感器数据均由 APM32 固件作为中介读取后通过 UART 返回。

**数据流（以 STATUS 命令为例）**：

```
① PC 通过 UART 发送 "STATUS\n"
② APM32 test_shell.c 收到命令
③ cmd_status() 调用 Mt6701ReadAngleDeg() → 软件 I²C 读取 MT6701 寄存器 0x03/0x04
④ MT6701 返回 14-bit 原始角度
⑤ 固件将 raw 和 angle_x100 打包为 "[TS] STATUS k=v ..."
⑥ APM32 通过 UART 返回给 PC
```

**两条数据采集路径的区别**：

| | 周期性 Debug 输出 | STATUS 命令 |
|---|---|---|
| 触发方 | 固件自动，每 500ms | PC 主动发送命令 |
| I²C 读取 | 主循环每 10ms 读一次，缓存于 `s_lastRawCount` | STATUS 命令触发一次新的 I²C 读取 |
| raw 数据 | `Mt6701GetLastRawDeg()` → 控制循环刚更新的缓存（≤10ms） | `Mt6701GetLastRawCount()` → **上一次** I²C 读的缓存，非本次实时 |
| angle 数据 | `motor_ctrl_get_angle()` → 控制循环刚读的值（映射+钳位） | `Mt6701ReadAngleDeg()` → **本次** STATUS 触发的新鲜 I²C 读取 |
| 输出格式 | `RUN state=... raw=...`（无前缀） | `[TS] STATUS raw=... angle_x100=...`（有 `[TS] ` 前缀） |

### 2.2 MT6701 传感器规格

| 参数 | 值 |
|------|-----|
| 型号 | MagnTek MT6701CT-STD-R |
| 分辨率 | **14-bit**（16384 counts / 360°） |
| 角度精度 | 典型 ±0.5°（出厂） |
| 通信接口 | I²C（MODE 引脚接地时） |
| I²C 地址 | **0x06**（7-bit） |
| 角度寄存器 | `0x03` ANGLE_H[7:0]（高 8 位）|
| | `0x04` ANGLE_L[7:2]（低 6 位）|
| 14-bit 角度组合 | `raw = (ANGLE_H << 6) \| (ANGLE_L >> 2)` |
| 供电电压 | 3.3V |

### 2.3 raw 与物理角度的换算

```
raw → 角度(°) = raw × 360.0 / 16384.0
```

| raw 值 | 对应角度 |
|--------|----------|
| 0 | 0.00° |
| 4096 | 90.00° |
| 8192 | 180.00° |
| 12288 | 270.00° |
| 16383 | 359.98° |

> raw 是 0~16383 的无符号整数，对应传感器磁铁 0°~360° 的绝对位置。**raw 值不受校准映射影响**，是精度测试的核心数据来源。

### 2.4 电机与机械结构

| 参数 | 值 |
|------|-----|
| 驱动板供电 | 24V DC 外部电源 |
| 有效行程 | 1.5° ~ 71.5°（校准后映射范围） |
| 机械止挡 | 正向 ~230° raw / 反向 ~162° raw（每次校准后可能变化） |

## 四、串口 Shell 数据采集接口

### 4.1 通信协议

| 参数 | 值 |
|------|-----|
| 物理接口 | UART1 (PA9 TX / PA10 RX) |
| 波特率 | **115200** |
| 数据位 | 8 |
| 校验位 | None |
| 停止位 | 1 |
| 命令格式 | ASCII 文本，以 `\n` 结尾（CR 忽略），大小写不敏感 |
| 回复前缀 | `[TS] ` — 所有 Shell 回复行均以此前缀开头，便于脚本过滤 |

### 4.2 两种数据输出源的区别

固件运行时串口同时输出**两类数据**：

| | 周期性 Debug 输出 | STATUS 命令回复 |
|---|---|---|
| 触发方式 | 自动，每 500ms | 手动发送 `STATUS\n` |
| 行格式 | `RUN state=... angle=... raw=...` | `[TS] STATUS k=v k=v ...` |
| 行前缀 | 无前缀 | `[TS] ` 前缀（脚本可过滤） |
| raw 来源 | `Mt6701GetLastRawDeg()`（浮点 °，来自上次 I²C 读缓存） | `Mt6701GetLastRawCount()`（整型 0-16383） |
| angle 来源 | `motor_ctrl_get_angle()`（映射后 + 范围钳位） | `Mt6701ReadAngleDeg()`（STATUS 命令触发的新鲜 I²C 读） |
| 用途 | 人工监看运行状态 | **脚本自动化精度数据采集** |

> **精度测试建议**：以 STATUS 命令的 `raw` 字段为数据源。`raw` 是 14-bit 原始整型值，直接从 MT6701 寄存器读取，无校准映射干扰，最能真实反映传感器精度。

### 4.3 STATUS 命令 — 字段字典

**命令**：`STATUS\n`

**回复示例**：
```
[TS] STATUS status=0 pwm=0 angle_x100=3000 target_x100=3000 fault=0 over=0 stall=0 timeout=0 state=1 uptime_ms=120000 raw=8192 calib_ok=1 sensor_err=0 ipropi_raw=150 current_ma=120 cal_min_x100=16209 cal_max_x100=24870 cal_span_x100=8661
```

**逐字段说明**：

#### 4.3.1 精度测试核心字段

| 字段 | 类型 | 示例 | 说明 |
|------|------|------|------|
| `raw` | uint16 | `8192` | MT6701 14-bit 原始计数值（0~16383）。**取自 `s_lastRawCount` 缓存**，即本次 STATUS 调用之前最后一次 I²C 读取的结果 |
| `angle_x100` | int32 | `3000` | 校准映射后的角度 × 100（centidegrees）。`3000` = 30.00°。**STATUS 命令触发一次新的 I²C 读取后计算得出** |

#### 4.3.1.1 `raw` 与 `angle_x100` 的时序关系（重要）

`cmd_status()` 的执行顺序决定了两个字段来自不同时刻的 I²C 读取：

```
cmd_status() 执行顺序：
  ① raw = Mt6701GetLastRawCount()        ← 读缓存 s_lastRawCount（上一轮的旧值）
  ② Mt6701ReadAngleDeg(&angle)           ← 触发新的 I²C 读，内部更新 s_lastRawCount
  ③ 打包回复: raw=旧值, angle_x100=新映射
```

因此**单次 STATUS 回复中，`raw` 和 `angle_x100` 不是同一时刻的采样**：

| | `raw` | `angle_x100` |
|---|---|---|
| I²C 读时刻 | 上一次 I²C 读（可能是控制循环约 10ms 前，也可能是上一次 STATUS 命令触发） | 本次 STATUS 命令触发的新鲜读取 |
| 来源 | `s_lastRawCount`（静态缓存变量） | `Mt6701ReadAngleDeg()` → 内部调用 `Mt6701ReadAngleRaw()` → 映射 |

**对连续采样（脚本轮询 STATUS 50 次）的影响**：

```
第 1 次 STATUS:  raw = 控制循环的旧值        angle = 本次 STATUS I²C 新鲜读
                 └─ 完成后 s_lastRawCount 被本次 STATUS 更新

第 2 次 STATUS:  raw = 第 1 次 STATUS 时 I²C 读的值  angle = 本次 STATUS I²C 新鲜读
                 └─ raw 和 angle 实际间隔 ≈ 两次 STATUS 之间的时间（约 10-20ms）

第 3 次 STATUS:  raw = 第 2 次 STATUS 时 I²C 读的值  angle = 本次 STATUS I²C 新鲜读
...
```

> **采集建议**：连续采样时第 1 次 STATUS 的 `raw` 值来自控制循环（可能已有 10ms 延迟），建议**丢弃第 1 次采样的 `raw` 值**，从第 2 次开始记录。第 2~N 次的 `raw` 与 `angle_x100` 之间的时间差仅为两次 STATUS 的发送间隔，在毫秒级可忽略。

### 4.4 数据采集命令

#### 4.4.1 MOVE — 移动电机到指定角度

**命令**：`MOVE <angle_x100> [speed]`

| 参数 | 说明 |
|------|------|
| `angle_x100` | 目标角度 × 100（centidegrees）。有效范围 `150` ~ `7150`（1.50° ~ 71.50°） |
| `speed`（可选） | 速度值（0~10000）。省略或为 0 时使用默认速度（PWM_SPEED_LIMIT = 10000） |

**示例**：
```
MOVE 3000        → 转到 30.00°（默认速度）
MOVE 1500 5000   → 以 50% 速度转到 15.00°
MOVE 7150        → 转到最大有效角度 71.50°
```

**回复**：
```
[TS] OK MOVE dispatched
```

**到达判定**：发送 MOVE 后，轮询 `STATUS` 等待 `status=2`（ARRIVED）。到达容差为 ±0.3°（`POSITION_ARRIVE_DEADBAND`）。

#### 4.4.2 其他辅助命令

| 命令 | 回复 | 用途 |
|------|------|------|
| `PING` | `[TS] PONG` | 心跳检测，验证 Shell 在线 |
| `STATUS` | `[TS] STATUS k=v ...` | 获取完整状态（见 4.3） |
| `STOP` | `[TS] OK STOP dispatched` | 紧急停止电机 |
| `CALIB` | `[TS] OK CALIB started` | 启动自动校准（摇臂须自由） |
| `FAULT_CLR` | `[TS] OK FAULT cleared` | 清除锁存的故障状态 |
| `HELP` | `[TS] commands: ...` | 列出所有可用命令 |

## 五、校准系统

### 5.1 校准的作用

MT6701 输出的是 0°~360° 的绝对角度（raw），但机械摇臂只在约 70° 范围内运动。校准的作用是：

1. 找到摇臂的两个机械止挡对应的 raw 值
2. 建立映射：**reverse stop raw → 0.00°**（最小映射角）、**forward stop raw → 75.00°**（最大映射角）
3. 将 raw 线性映射到操作角度 `angle_x100`

### 5.2 校准流程

```
CALIB 命令
  → FWD_SETTLE (30 cycle, ~300ms)    PWM=+3000, 让电机启动
  → FWD_SEARCH                      PWM=+3000, 前进直到碰到止挡
       → raw 连续 80 次变化 ≤ 2 → 判定到达正向止挡
  → REV_SETTLE (30 cycle, ~300ms)    PWM=-3000, 反向启动
  → REV_SEARCH                      PWM=-3000, 后退直到碰到止挡
       → raw 连续 80 次变化 ≤ 2 → 判定到达反向止挡
  → 保存映射到 Flash（CRC 校验）
  → DONE
```

- 每个阶段最大 1200 cycles（~12s），超时则校准失败
- 校准数据保存在 Flash 末页（0x0801FC00），上电时自动加载，不受固件重烧影响

### 5.3 校准数据的含义

在 STATUS 中看到的校准字段：
```
cal_min_x100=16209   → 反向止挡 raw = 162.09°
cal_max_x100=24870   → 正向止挡 raw = 248.70°
cal_span_x100=8661   → 行程 = 86.61°
```

这表示：当 raw 从 162.09° 变到 248.70° 时，`angle_x100` 从 0 线性映射到 7500（75.00°）。

### 5.4 测试前校准检查

每次上电后应先确认校准状态：

```
STATUS
```

检查：
- `calib_ok=1` ✅
- `enc_err=0`（周期性输出中） ✅
- 当前 `angle_x100` 在 0~7500 范围内 ✅

如果 `calib_ok=0` 或校准数据异常，发送 `CALIB` 重新校准。

## 六、数据采集脚本接口规范

### 6.1 串口连接

```
Python (pyserial):
  port     = COM3 (Windows) 或 /dev/cu.wchusbserial* (macOS)
  baudrate = 115200
  bytesize = 8
  parity   = 'N'
  stopbits = 1
  timeout  = 0.5 (读超时)
```

### 6.2 命令-回复交互模式

每条命令以 `\n` 结尾。回复可能混合有周期性 debug 输出，脚本应**按 `[TS] ` 前缀过滤**：

```python
def send_cmd(ser, cmd):
    ser.write((cmd + '\n').encode())
    ser.flush()
    while True:
        line = ser.readline().decode().strip()
        if line.startswith('[TS] '):
            return line[5:]  # 去掉 [TS] 前缀
```

### 6.3 STATUS 回复解析

```python
def parse_status(reply):
    """解析 'STATUS k=v k=v ...' 为 dict"""
    parts = reply.split()
    result = {}
    for p in parts[1:]:  # 跳过 'STATUS' 本身
        if '=' in p:
            k, v = p.split('=', 1)
            result[k] = int(v)  # 所有值均为整数
    return result
```

### 6.4 单采样点采集流程

```
1. 发 MOVE <angle_x100>
   → 收到 '[TS] OK MOVE dispatched'

2. 轮询 STATUS，直到 status == 2 (ARRIVED)
   → 每次 STATUS 延时约 50-100ms
   → 超时约 10s（如电机卡住）

3. 到达后，延迟 200ms 让电机完全稳定

4. 循环 N 次（N = 50）：
   → 发 STATUS
   → 记录 raw 值
   → 延时约 10ms（给一次 I²C 读取腾时间）

5. 处理下一个角度
```

### 6.5 伪代码

```python
SAMPLE_POINTS = [150, 650, 1150, 1650, ..., 7150]  # 1.5°~71.5°, 步长 5°
SAMPLES_PER_POINT = 50

for angle_x100 in SAMPLE_POINTS:
    send_cmd(ser, f'MOVE {angle_x100}')
    wait_for_arrived(ser, timeout_s=10)
    time.sleep(0.2)

    raw_values = []
    for i in range(SAMPLES_PER_POINT):
        reply = send_cmd(ser, 'STATUS')
        data = parse_status(reply)
        raw_values.append(data['raw'])
        time.sleep(0.01)

    # 计算统计量
    mean = statistics.mean(raw_values)
    std = statistics.stdev(raw_values)
    peak_to_peak = max(raw_values) - min(raw_values)

    record(angle_x100, mean, std, peak_to_peak, raw_values)
```

## 七、常见问题与排查

| 现象 | 可能原因 | 排查/解决 |
|------|----------|-----------|
| `PING` 无回复 | 固件未烧录 TEST_BUILD 版本 | 确认 Define 中有 `TEST_BUILD`，`test_shell.c` 已加入工程 |
| `enc_err > 0` | I²C 通信异常 | 检查 PB10/PB11 接线，确认上拉电阻和供电 |
| `enc_err` 持续增长 | MT6701 完全无响应 | 同 I²C 排查，检查 MODE 引脚电平 |
| `calib_ok=0` | 无有效校准数据 | 发送 `CALIB`，确保摇臂自由且能在两个方向碰到止挡 |
| `fault=1` + `status=3` | 保护触发 | 发送 `FAULT_CLR` 清除，检查是否机械卡死或电流异常 |
| MOVE 后 status 始终不是 2 | 电机无法到达目标 | 检查 `current_ma`、`stall` 标志，检查驱动板 24V 供电 |
| `current_ma` 异常偏高（> 500mA 空闲） | IPROPI ADC 引脚配置错误 | 对比电路图确认 PA3（非 PA0），检查 `board_config.h` |
| `raw` 恒定不变 | MT6701 读不到数据或磁铁未安装 | 确认 `enc_err=0`，确认径向磁铁正确安装在电机轴上 |
